# JTDOS SYSTEM BIBLE

## 1. Project Name

JTDOS  
Japan Tourism Driver Operating System

## 2. Core Vision

JTDOS is an AI-powered execution operating system for Japan tourism transportation services.

It connects AI travel sales agents with real-world driver dispatch, vehicle booking, airport transfer, charter tour, and local tourism execution.

The goal is not only to build a website, but to build an AI-callable tourism execution infrastructure.

## 3. Core Concept

AI Agent is the brain.  
JTDSS is the execution engine.  
JTDOS is the operating system connecting both.

User inquiry  
-> AI understands travel needs  
-> AI asks missing questions  
-> AI generates quote  
-> AI generates itinerary  
-> AI creates structured order  
-> JTDSS receives order  
-> Drivers receive dispatch  
-> Service is executed in the real world

## 4. Main Service Types

1. Airport pickup

4. Full-day charter

## 7. Core Value

JTDOS turns real Japan tourism transportation experience into structured AI tools.

It allows any AI agent, website, or partner to generate Japan travel orders and send them into the JTDSS execution system.

## 8. Alpha 0.1 Goal

The first version must complete one booking request and execution preparation loop:

Customer inquiry  
-> AI extracts order information  
-> AI generates quote  
-> AI creates booking request  
-> Operator Liu Xiang confirms final price, vehicle availability, driver arrangement, and deposit  
-> Order is sent or synchronized to JTDSS  
-> Driver can receive or view the confirmed order

## 9. What We Do Not Build in Alpha 0.1

Do not build full OTA functions.  
Do not build full payment system.  
Do not build hotel booking.  
Do not build flight booking.  
Do not build complex driver ranking.  
Do not support all Japan routes at the beginning.

Alpha 0.1 only proves the booking request to operator-confirmed execution loop.

## 10. Core Principle

Do not build a chatbot only.

Build an AI-to-real-world execution system.
